== How to Become a Maintainer

We welcome contributors who demonstrate:

1. **Sustained Contributions**: Regular, high-quality PRs over 3+ months
2. **Technical Excellence**: Deep understanding of the codebase
3. **Community Engagement**: Helpful code reviews, issue triage, documentation
4. **Alignment**: Share project values and vision
5. **Reliability**: Consistent availability and responsiveness

=== Process

1. **Nomination**: Any current maintainer can nominate a contributor
2. **Discussion**: Maintainers discuss the nomination privately
3. **Consensus**: All current maintainers must approve (unanimity required)
4. **Invitation**: Nominee is invited to join the maintainer team
5. **Onboarding**: New maintainer receives access and orientation

== Stepping Down

Maintainers may step down at any time by:

1. Notifying other maintainers
2. Transferring active responsibilities
3. Moving to emeritus status (optional, with recognition)

We deeply appreciate all maintainer contributions and respect life circumstances that may require stepping back.

== Maintainer Meetings

* **Frequency**: As needed (typically monthly or quarterly)
* **Format**: Asynchronous (GitHub Discussions) or synchronous (video call)
* **Agenda**:
  - Project roadmap
  - Major decisions
  - Community health
  - Security updates
  - Release planning

== Decision-Making Process

=== Minor Decisions
* Single maintainer can decide
* Examples: Bug fixes, documentation updates, dependency updates

=== Major Decisions
* Consensus among all maintainers
* Examples: Architecture changes, new features, breaking changes, governance updates

=== Conflict Resolution
1. Discussion in private maintainer channel
2. Attempt to reach consensus
3. If consensus fails, vote (simple majority)
4. Project lead has tie-breaking vote (used sparingly)

== Maintainer Privileges

* Write access to main repository
* Ability to merge pull requests
* Access to release process
* Access to security reports
* Ability to modify GitHub settings (labels, milestones, etc.)
* Invitation to private maintainer discussions

== Code of Conduct Enforcement

Maintainers are responsible for enforcing the Code of Conduct:

1. **First Violation**: Private warning
2. **Second Violation**: Public warning
3. **Third Violation**: Temporary ban
4. **Severe Violations**: Immediate permanent ban

Maintainers must recuse themselves from enforcement decisions where they have a conflict of interest.
