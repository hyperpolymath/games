== Our Ethical Framework

=== 1. Proximity of Harm

We accept responsibility based on how **foreseeable** the harm is:

* ‚úÖ **Highly foreseeable**: BSOD used for tech support scams
  - **Our responsibility**: Document this risk, add warnings
* ‚ö†Ô∏è **Somewhat foreseeable**: User gets angry, damages property
  - **Shared responsibility**: We warn about pranks, user chooses targets
* ‚ùå **Unforeseeable**: Multi-step chains leading to unpredictable outcomes
  - **Not our responsibility**: Cannot control all possible futures

=== 2. Primary Design Intent

* **Not designed for harm** - Primary purpose is humor/education
* **Requires modification for weaponization** - Must add deception, modify UI
* **Transparent implementation** - Open source, clearly documented as joke
* **Harm signals included** - Humorous error messages make scam use less effective

=== 3. Reasonable Mitigations

What we've done to reduce harm:

* ‚úÖ **Obvious humor signals** - `COFFEE_NOT_FOUND`, `NPM_INSTALL_TIMEOUT`
* ‚úÖ **No system access** - Purely visual, doesn't touch actual OS
* ‚úÖ **Transparent code** - Open source, auditable
* ‚úÖ **Clear documentation** - Explicitly labeled as humor tool
* ‚úÖ **Educational framing** - RSR/TPCF compliance, security practices
* ‚úÖ **Responsible disclosure** - RFC 9116 security.txt

== Known Risks & Warnings

=== ‚ö†Ô∏è Tech Support Scams

**Risk**: Scammers could use this BSOD UI to convince users their computer is broken.

**Mitigation**:
* Humorous error messages make it obviously fake to alert users
* No phone numbers or "call now" messaging in our code
* Educational value teaches people to recognize fake errors

**Warning**: Do not modify this to remove humor signals for deceptive purposes.

=== ‚ö†Ô∏è Psychological Distress

**Risk**: Non-technical users may genuinely panic when seeing a BSOD.

**Mitigation**:
* Clear disclaimer that this is humor/prank software
* Documentation emphasizes consent and appropriate contexts

**Warning**: Do not use this on people who:
* Have anxiety disorders
* Are non-technical and might genuinely panic
* Are in high-stress situations (exams, job interviews, emergencies)
* Did not consent to being pranked

=== ‚ö†Ô∏è Professional Contexts

**Risk**: Using this in professional/workplace settings could have consequences.

**Warning**: Do not use this:
* On employer-owned devices without permission
* In critical business meetings or presentations
* On shared/public computers
* In contexts where it could be mistaken for actual system failure

== What We Cannot Control

**We acknowledge:**

1. **Future modifications** - Someone could fork this and remove humor signals
2. **Unforeseeable chains** - Infinite possible consequences we cannot predict
3. **Cultural differences** - What's funny in one context may be harmful in another
4. **Individual reactions** - Cannot control how each person responds

**We accept** that perfect safety is impossible and that creating nothing also causes harm (no tools, no innovation, no progress).
